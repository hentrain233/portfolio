# Balance Config Map

下面是目前已拆分到 `data/` 的可调配置分类。后续做数值平衡时，优先改这些文件。

## 1) 核心战斗与流程参数
- 文件：`data/tuning.js`
- 内容：
  - 玩家基础：初始能量上限、每回合抽牌、初始最大生命
  - 调试能量
  - 战斗倍率：虚弱/易伤乘区
  - 基础值：打击基础伤害、防御基础格挡
  - 表现节奏：多段攻击特效间隔
  - 篝火回血比例
  - 战斗胜利金币（普通/BOSS/蛇事件战）

## 2) 概率与价格区间
- 文件：`data/probabilities.js`
- 内容：
  - 地图节点权重、事件转战斗概率
  - 奖励卡稀有度权重、升级卡出现概率
  - 稀有事件概率、水井失败概率
  - 商店卡牌/遗物稀有度权重
  - 商店卡牌/遗物按稀有度价格区间

## 3) 商店结构与服务规则
- 文件：`data/shopConfig.js`
- 内容：
  - 商店固定层位置、地图总节点数
  - 商店库存数量（卡牌/遗物）
  - 服务基础价格与每次使用涨价步进

## 4) 事件定义（文案+选项+数值）
- 文件：`data/eventDefs.js`
- 内容：
  - 事件池（普通/稀有）
  - 事件标题
  - 事件描述
  - 事件选项文案
  - 事件数值（例如水井上限、失败基础伤害、蛇事件代价、祝福金币事件收益/代价）

## 5) 卡牌、祝福、敌人、角色
- 卡牌模板与升级值：`data/card-config.js`
- 祝福与遗物定义：`data/blessing-config.js`
- 敌人数值与行为：`data/enemies.js`
- 角色起始牌组：`data/characters.js`

## 建议
- 新增系统时，先决定它属于哪一类（概率/基础数值/内容定义），再放进对应 `data/*.js`。
- `game.js` / `systems/*` 尽量只保留“读取配置并执行”的逻辑，不再新增硬编码数值。
